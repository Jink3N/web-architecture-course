🔗 URL, URI, URN

Identificatori universali per risorse nel web.

🎯 Le Tre Tipologie di Identificatori

Nel web esistono tre tipi fondamentali di identificatori per risorse, spesso confusi ma con scopi diversi.

🌐 URL - Uniform Resource Locator

L'indirizzo di una risorsa

  • Funzione: Specifica DOVE trovare una risorsa
  • Include: Protocollo, server, percorso
  • Esempio: https://www.example.com/pages/about.html
  • Caratteristica: Può cambiare se la risorsa si sposta

Analogia: Come un indirizzo fisico - ti dice dove andare

🆔 URN - Uniform Resource Name

Il nome permanente di una risorsa

  • Funzione: Identifica COSA è una risorsa (nome unico)
  • Include: Namespace, identificatore specifico
  • Esempio: urn:isbn:9780123456789
  • Caratteristica: Immutabile, non dice dove trovarla

Analogia: Come il codice fiscale - identifica univocamente

🔗 URI - Uniform Resource Identifier

Il concetto generale che include sia URL che URN. Ogni URL è un URI, ogni URN è un URI, ma non viceversa.

Formula: URI = URL ∪ URN (URI è l'insieme che contiene entrambi)

🧩 Anatomia di un URL

Un URL completo è composto da diverse parti, ognuna con uno scopo specifico.

1

🔒 Schema/Protocollo

https://www.example.com/path?query=value#section

  • http/https: Web standard
  • ftp: File transfer
  • mailto: Email addresses
  • file: File locali
2

🌍 Authority/Host

https://www.example.com/path?query=value#section

  • Subdomain: www, api, cdn
  • Domain: example.com
  • Port: :8080 (opzionale)
  • Credentials: user:pass@ (deprecato)
3

📁 Path

https://www.example.com/path/to/resource?query=value#section

  • Hierarchical: Struttura simile a folder
  • Case sensitive: /Path ≠ /path
  • Encoding: Spazi diventano %20
  • Trailing slash: /path/ vs /path può essere diverso
4

❓ Query String

https://www.example.com/path?param1=value1¶m2=value2#section

  • Key-value pairs: Separati da &
  • URL encoding: Caratteri speciali encoded
  • Optional: Può essere assente
  • Order: Generalmente non importa
5

🎯 Fragment/Hash

https://www.example.com/path?query=value#section-id

  • Client-side: Non inviato al server
  • Page navigation: Anchor links
  • SPA routing: Usato per client routing
  • State: Può contenere application state

📚 Esempi di URN

I URN forniscono identificazione permanente senza specificare la location.

📖 ISBN Books

urn:isbn:9780123456789

Identifica univocamente un libro specifico a livello mondiale, indipendentemente da dove si trova.

Namespace: isbn

📄 DOI Documents

urn:doi:10.1000/182

Digital Object Identifier per articoli scientifici, paper di ricerca, documenti accademici.

Namespace: doi

🌐 UUID Universal

urn:uuid:6e8bc430-9c3a-11d9-9669-0800200c9a66

Universally Unique Identifier per qualsiasi tipo di risorsa digitale.

Namespace: uuid

🏛️ Legal Documents

urn:lex:it:stato:legge:2023-01-15;42

Identificazione standard per documenti legali e normativi.

Namespace: lex

🔧 URL Encoding e Best Practices

📝 URL Encoding

Caratteri speciali devono essere encoded per sicurezza.

  • Spazi: %20 o + (in query)
  • Non-ASCII: UTF-8 encoding
  • Reserved chars: ? & = # % etc.
  • Unsafe chars: < > " { } | ^ ~

🔗 URL Structure Best Practices

Design intuitivo e SEO-friendly.

  • Descriptive: /products/laptops vs /cat/123
  • Lowercase: Evita case sensitivity issues
  • Hyphens: word-separator invece di underscores
  • Consistent: Schema URL coerente

⚠️ Common URL Problems

Errori frequenti nella gestione URL.

  • Length limits: 2000+ chars problematici
  • Sensitive data: Mai password in URL
  • Special characters: Encoding mancante
  • Trailing slash: Inconsistenza comportamento

🔒 Security Considerations

Sicurezza nella gestione URL.

  • No secrets: Mai token/password in URL
  • Validation: Sempre validare input URL
  • Canonicalization: Normalize before processing
  • Access control: Verify permissions per path

🌍 URL Internationalization

Gestione di URL con caratteri non-ASCII e domini internazionali.

🌐 IDN - International Domain Names

Punycode encoding per domini non-ASCII

// Domain con caratteri speciali
例え.テスト → xn--r8jz45g.xn--zckzah

// URL completo
https://例え.テスト/ページ → 
https://xn--r8jz45g.xn--zckzah/%E3%83%9A%E3%83%BC%E3%82%B8

🔤 UTF-8 URL Encoding

Caratteri Unicode in path e query

// Caratteri accentati
café → caf%C3%A9

// Emoji in URL (tecnicamente possibile)
https://example.com/🏠 → 
https://example.com/%F0%9F%8F%A0

📱 Modern URL Handling

Browser moderni gestiscono Unicode

// JavaScript URL API
const url = new URL('https://example.com/café');
console.log(url.pathname); // '/café'
console.log(url.href); // 'https://example.com/caf%C3%A9'

// Encoding/decoding
encodeURIComponent('café'); // 'caf%C3%A9'
decodeURIComponent('caf%C3%A9'); // 'café'

🎨 URL Beautification

User-friendly URLs per miglior UX

// Before: ugly URLs
/article.php?id=123&cat=tech

// After: clean URLs
/articles/tech/my-awesome-article-title

// SEO-friendly slugs
"My Awesome Article!" → "my-awesome-article"

🎯 Best Practices URL Design

User Experience: URL dovrebbero essere leggibili, prevedibili e condivisibili facilmente.

SEO Optimization: Include keyword rilevanti, usa struttura gerarchica logica, evita parametri quando possibile.

Technical: Implementa redirect 301 per URL obsoleti, usa canonical URLs, gestisci trailing slash consistentemente.

Security: Valida sempre input URL, evita injection attacks, non esporre informazioni sensibili.
