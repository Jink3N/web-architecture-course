📱 Single Page Application

Una SPA è un'applicazione web che interagisce dinamicamente con l'utente riscrivendo la pagina corrente piuttosto che caricare intere nuove pagine dal server.

📜 Storia delle Single Page Applications

Da AJAX (2005) a framework moderni. Gmail (2004) fu una delle prime SPA mainstream, seguita da Google Maps.

A cosa serve: Creare applicazioni web che si comportano come desktop apps con user experience fluida e responsive.

Quando usarla: User experience premium richiesta, applicazioni interattive, real-time data, dashboards complesse.

🏗️ Architettura di una SPA

1

🚀 Caricamento Iniziale

Il browser scarica l'HTML minimale, CSS e bundle JavaScript dell'intera applicazione.

  • HTML skeleton con div root
  • Bundle JS contenente l'app
  • Fogli di stile CSS
  • Asset statici necessari
2

🎮 Bootstrap dell'App

Il framework JavaScript inizializza l'applicazione e configura il router client-side.

  • Mounting dei componenti
  • Inizializzazione del router
  • Setup dello state management
  • Configurazione dei servizi
3

🔄 Navigazione Dinamica

JavaScript intercetta navigazione e aggiorna solo il contenuto necessario senza ricaricare la pagina.

  • History API per URL
  • Rendering condizionale
  • Gestione dello stato
  • Transizioni fluide
4

📡 Comunicazione API

L'app comunica con il backend via AJAX/Fetch per dati JSON, aggiornando UI dinamicamente.

  • Chiamate HTTP asincrone
  • Parsing JSON responses
  • Update reattivo dell'UI
  • Error handling

⚡ Vantaggi vs ⚠️ Svantaggi

🏎️ Performance Dinamica

Dopo il caricamento iniziale, la navigazione è istantanea e fluida.

  • No page reload
  • Caching intelligente
  • Lazy loading
  • Ottimizzazioni runtime

🎨 UX Premium

Esperienza utente simile alle app native desktop/mobile.

  • Transizioni smooth
  • Feedback immediato
  • Stato persistente
  • Interazioni ricche

🔍 SEO Complesso

Indicizzazione difficile per contenuto generato dinamicamente.

  • JS-rendered content
  • URL routing complesso
  • Meta tags dinamici
  • Server-side rendering necessario

⏱️ First Load Pesante

Caricamento iniziale lento per bundle JavaScript grandi.

  • Bundle size elevato
  • Time to Interactive alto
  • Code splitting necessario
  • Progressive loading complesso

🛠️ Framework Moderni

⚛️ React

Libreria di Facebook: Component-based, Virtual DOM, ecosistema vastissimo.

Ideale per: App complesse, team grandi, ecosistema maturo

🅰️ Angular

Framework Google: TypeScript-first, dependency injection, CLI potente.

Ideale per: Enterprise, applicazioni grandi, team strutturati

💚 Vue.js

Framework progressivo: Curva apprendimento dolce, template intuitivi.

Ideale per: Progetti medi, team misti, migrazione graduale

⚡ Svelte

Compilatore: No runtime overhead, bundle piccoli, performance native.

Ideale per: Performance critiche, bundle size limitato

🎯 Quando Scegliere SPA vs MPA?

Scegli SPA quando: UX interattiva prioritaria, app complessa, separazione frontend/backend, sviluppo mobile parallelo.

Scegli MPA quando: SEO critico, contenuto principalmente statico, semplicità di sviluppo, supporto browser legacy.

📧 Curiosità: Gmail "Project Caribou"

Gmail fu sviluppata in segreto come "Project Caribou" e quasi chiamata "Javelin". Fu una delle prime applicazioni a dimostrare che il web poteva competere con il desktop!

Innovazioni di Gmail: AJAX pesante, interfaccia ricca, gestione email rivoluzionaria, conversation threading.
