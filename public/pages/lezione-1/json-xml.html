📄 JSON vs XML

Due formati di serializzazione dati con filosofie e casi d'uso differenti.

📜 Storia

XML nasce nel 1998 (W3C) per documenti strutturati e interoperabilità enterprise. JSON (Douglas Crockford, 2001) emerge come formato leggero per il web dinamico.

🧩 Struttura Base

JSON

{
  "user": {
    "id": 123,
    "name": "Ada",
    "roles": ["admin", "editor"],
    "active": true
  }
}

Oggetti, array, scalari nativi.

XML

<user id="123" active="true">
  <name>Ada</name>
  <roles>
    <role>admin</role>
    <role>editor</role>
  </roles>
</user>

Elementi, attributi, gerarchie flessibili.

🔍 Differenze Chiave

JSON

Sintassi compatta, parsing veloce nativo in JS.

XML

Supporta namespaces, validazione schema forte (XSD).

JSON Limiti

Nessun commento, niente namespaces, tipizzazione meno rigorosa.

XML Limiti

Verboso, parsing più lento, overhead maggiore.

🛡️ Sicurezza

JSON Rischi

  • Injection se eval usato
  • Prototype pollution
  • Exfiltration via JSONP (storico)

XML Rischi

  • XXE (Entity Expansion)
  • Billion laughs attack
  • XPath injection

Mitigazioni JSON

  • Usare JSON.parse
  • Validare tipi
  • Sanitizzare input

Mitigazioni XML

  • Disabilitare DTD
  • Parser sicuri
  • Limiti dimensione

🔄 Conversione

XML → JSON

<item price="10">Book</item>
{
  "item": {
    "_text": "Book",
    "price": 10
  }
}

Attributi ↔ proprietà.

Array Heuristic

<tags>
  <tag>a</tag>
  <tag>b</tag>
</tags>
{"tags": ["a", "b"]}

Elementi ripetuti → array.

Date Handling

{"createdAt": "2024-09-10T12:30:00Z"}

Stringhe ISO 8601 standard.

Schema XML

<xs:element name="price" type="xs:decimal"/>

Tipizzazione forte.

⚙️ Performance

JSON Parsing

Lineare, supporto nativo motori JS.

XML Streaming

Con parser SAX/StAX gestisce grandi file.

Overhead XML

Tag di apertura/chiusura aumentano dimensione.

Compressione

Gzip riduce gap ma JSON resta più piccolo.

🧪 Validazione

JSON Schema

{
  "type": "object",
  "properties": {"id": {"type": "integer"}},
  "required": ["id"]
}

XSD

<xs:element name="id" type="xs:int"/>

Relax NG

<element name="user">
  <element name="id" type="int"/>
</element>

Valori Null

{"middleName": null}

XML: elemento mancante vs vuoto.

🎯 Quando Usare Cosa

JSON: API web, mobile, microservizi, payload leggeri.

XML: Sistemi legacy, documenti complessi, SOAP, standard enterprise.

Entrambi: Conversione in pipeline ETL, interoperabilità tra sistemi.

💡 Curiosità

JSON inizialmente non aveva uno standard formale (RFC 4627 arrivato solo nel 2006). XML invece nasce già con endorsement enterprise.
