🆔 OpenID Connect (OIDC)

Layer di identità sopra OAuth 2.0 per Single Sign-On moderno.

🔍 Perché OIDC?

OAuth Limite

  • Nessun standard user info
  • Nessuna identità garantita

OIDC Aggiunge

  • ID Token firmato
  • UserInfo endpoint

🧩 Componenti

ID Token

{
 "iss":"https://idp",
 "sub":"123456",
 "aud":"client_id",
 "exp":1712345678,
 "nonce":"xyz"
}

Access Token

Come OAuth (per API).

Refresh Token

Per sessione lunga.

UserInfo

GET /userinfo
Authorization: Bearer <token>

⚙️ Flow Tipico

1

Auth Request

response_type=code&scope=openid email.

2

Login

IdP autentica.

3

Code

Redirect secure.

4

Token

Ottieni id_token + access.

5

Validazione

Firma, exp, nonce, aud.

6

UserInfo

Dati profilo extra.

🛡️ Validazione ID Token

Controlli

  • Firma JWS
  • iss atteso
  • aud = client
  • exp futuro

Rischi

  • Kid spoofing
  • Alg none attack
  • Nonce mancante
  • Replay token

Mitigazioni

  • JWKS caching
  • Enforce alg
  • Nonce store
  • Timestamps stretti

Errori

  • Usare access per identità
  • Niente PKCE
  • Scopes eccessivi
  • Salvare id_token non cifrato

🔑 Scopes Standard

openid

Obbligatorio, abilita OIDC.

profile

Nome, cognome, ecc.

email

Email + verifica.

offline_access

Refresh token.

🎯 Sintesi

OIDC fornisce identità sicura e interoperabile sopra OAuth per SSO moderno.

🌍 Ecosistema

Base per login Google, Azure AD, Auth0, Keycloak, Okta.
