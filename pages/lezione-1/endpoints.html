🎯 API Endpoints

Punti di accesso stabili e semantici alle risorse dell'API.

🔍 Cos'è un Endpoint?

Un endpoint è una combinazione di URI + Metodo HTTP che identifica un'operazione consentita su una risorsa. Progettazione coerente = API intuitiva e discoverable.

🧱 Principi di Naming

Sostantivi plurali

/users
/users/{id}
/orders/{orderId}

Niente verbi nell'URI.

Relazioni nidificate

/users/{id}/orders
/users/{id}/orders/{orderId}

Solo se relazione forte.

Filtri via query

/users?role=admin&active=true
/orders?from=2024-01-01&to=2024-01-31

No endpoint separati per filtri.

Azioni rare

POST /invoices/{id}/pay
POST /users/{id}/reset-password

Solo quando non CRUD.

⚙️ Pattern CRUD

C

Create

POST /resources

R

Read

GET /resources/{id}

U

Update

PUT /resources/{id} o PATCH /resources/{id}

D

Delete

DELETE /resources/{id}

📦 Paginazione & Filtri

Paginazione

/users?page=2&limit=25
Link: <...page=3>; rel="next"

Preferire link HATEOAS.

Field Selection

/users?fields=id,name,email

Riduce payload.

Sorting

/orders?sort=-createdAt,status

- = desc, default asc.

Filtering

/products?price[lt]=100&stock[gt]=0

Operatore nei parametri.

🔐 Sicurezza Endpoint

Least Privilege

Scope/token limitati per uso minimo necessario.

Rate Limiting

Protezione da abuso: 429 Too Many Requests.

Input Validation

Validare path, query, body; evitare injection.

Idempotency Keys

Per POST sensibili: header Idempotency-Key.

🧪 Idempotency Key Esempio

Richiesta

POST /payments
Idempotency-Key: 550e8400-e29b
{
  "amount": 1000,
  "currency": "EUR"
}

Risposta 1

201 Created
{ "paymentId": "p_123" }

Risposta 2 (retry)

200 OK
{ "paymentId": "p_123" }

Error

409 Conflict
Reason: Duplicate operation

🛠️ Esempio Strutturato

Resource Model

GET /users/123
{
  "id": "123",
  "name": "Ada Lovelace",
  "links": [
    {"rel": "self", "href": "/users/123"},
    {"rel": "orders", "href": "/users/123/orders"}
  ]
}

Error Model

{
  "error": "validation_error",
  "details": [{"field": "email", "message": "invalid"}]
}

Partial Update

PATCH /users/123
{ "name": "Ada L." }

Bulk Ops

POST /users/bulk
[{"name": "A"}, {"name": "B"}]

🎯 Best Practices

URI semantici, versionamento chiaro, errori consistenti, sicurezza difensiva, documentazione OpenAPI aggiornata.
