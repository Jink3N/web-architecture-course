🗝️ API Keys

Identificatori statici per accesso programmatico a servizi.

📌 Caratteristiche

Semplici

Stringa segreta

  • Header o query
  • Nessuna identità utente
  • Rotazione manuale
  • Scope limitati (se implementati)

Limitazioni

Non standard

  • Niente OAuth flow
  • No user consent
  • Revoca grossolana
  • Rischio leakage

🔧 Invio Tipico

HTTP Header

GET /v1/data\nX-API-Key: sk_live_123

Authorization

Authorization: ApiKey sk_live_123

Query (sconsigliato)

?api_key=sk_live_123

Esposto in log / cache.

cURL

curl -H "X-API-Key: sk_live_123" https://api.example.com/v1/users

🛡️ Sicurezza

Protezione

  • Solo HTTPS
  • Scope per key
  • Rate limiting
  • IP allowlist

Rischi

  • Commit accidentali
  • Key reuse
  • Nessun context
  • Rotazione rara

Mitigazioni

  • Scanner secrets CI
  • TTL ridotto
  • Prefix & checksum
  • Audit logging

Uso improprio

  • Auth utenti
  • Mobile embedded
  • Client pubblici
  • No revoca granulare

🔄 Rotazione

Dual Key

1. Crea nuova key\n2. Distribuisci\n3. Verifica traffico\n4. Revoca vecchia

Formato

pk_live_xxx (public)\nsk_live_xxx (secret)

Storage

ENV VAR + secret manager

Revoca

PATCH /keys/{id} {"active": false}

🎯 Sintesi

Usare API Keys per server-to-server semplice, non per delega utente o accesso ad alto rischio.

📚 Curiosità

Le prime API pubbliche usarono solo API keys plain prima dell'adozione di OAuth.
