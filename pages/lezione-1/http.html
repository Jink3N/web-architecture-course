🌐 Protocollo HTTP

HTTP è il protocollo fondamentale che abilita la comunicazione nel World Wide Web.

📜 Storia del Protocollo HTTP

Inventato da Tim Berners-Lee nel 1989, standardizzato nel 1991. HTTP/2 nel 2015, HTTP/3 nel 2022.

1989-1991: HTTP/0.9

Il protocollo primitivo: Solo metodo GET, nessun header, solo HTML. "dammi questa pagina e chiudi connessione".

1996: HTTP/1.0

Prima standardizzazione: Headers, status codes, metodi POST/HEAD, supporto per diversi tipi di contenuto.

1997: HTTP/1.1

Connessioni persistenti: Keep-alive, chunked encoding, caching migliorato, Host header obbligatorio.

2015: HTTP/2

Rivoluzione binaria: Multiplexing, server push, compressione headers, prioritizzazione stream.

2022: HTTP/3

QUIC Protocol: UDP-based, riduce latenza, connection migration, crittografia integrata.

🔧 Metodi HTTP Essenziali

📖 Metodi Sicuri (Safe)

GET - Leggi Risorsa

  • Idempotente: Stessa richiesta = stesso risultato
  • Cacheable: Può essere memorizzato in cache
  • Sicuro: Non modifica lo stato del server
  • Uso: Recuperare dati, pagine web, API read

HEAD - Solo Headers

  • Come GET ma senza body di risposta
  • Utile per metadati e validazione cache
  • Controllo esistenza risorsa

OPTIONS - Capabilities

  • Scopre metodi supportati dal server
  • Usato nei preflight CORS
  • Negoziazione capabilities

✏️ Metodi di Modifica

POST - Crea/Processa

  • Non idempotente: Ogni richiesta può avere effetti diversi
  • Non cacheable: Di default non memorizzabile
  • Flessibile: Creazione, processing, upload
  • Uso: Form submit, API create, file upload

PUT - Sostituisci Completamente

  • Idempotente: Stesso risultato se ripetuto
  • Sostituisce intera risorsa
  • Crea se non esiste

PATCH - Modifica Parziale

  • Modifica solo campi specificati
  • Più efficiente di PUT per update piccoli
  • Richiede formato patch definito

DELETE - Rimuovi Risorsa

  • Idempotente: Eliminare qualcosa di già eliminato è OK
  • Può restituire 204 No Content
  • Soft vs hard delete

📊 Status Codes Fondamentali

2xx

✅ Successo

  • 200 OK: Richiesta completata con successo
  • 201 Created: Nuova risorsa creata
  • 204 No Content: Successo senza body di risposta
  • 206 Partial Content: Range request parziale
3xx

🔄 Redirection

  • 301 Moved Permanently: Risorsa spostata definitivamente
  • 302 Found: Redirection temporaneo
  • 304 Not Modified: Cache valida, non scaricare
  • 307 Temporary Redirect: Mantieni metodo HTTP
4xx

❌ Errori Client

  • 400 Bad Request: Sintassi richiesta non valida
  • 401 Unauthorized: Autenticazione richiesta
  • 403 Forbidden: Autorizzazione negata
  • 404 Not Found: Risorsa non esistente
  • 429 Too Many Requests: Rate limiting
5xx

💥 Errori Server

  • 500 Internal Server Error: Errore generico server
  • 502 Bad Gateway: Proxy/gateway error
  • 503 Service Unavailable: Server temporaneamente indisponibile
  • 504 Gateway Timeout: Timeout upstream

📋 Headers Cruciali

🗂️ Content Headers

Content-Type: Tipo MIME del body

Content-Length: Dimensione in bytes

Content-Encoding: Compressione (gzip, br)

Essenziali per interpretare i dati

🏪 Cache Headers

Cache-Control: Direttive di caching

ETag: Identifier univoco versione

Last-Modified: Data ultima modifica

Ottimizzano performance e bandwidth

🔐 Security Headers

Authorization: Credenziali di accesso

X-Frame-Options: Protezione clickjacking

Strict-Transport-Security: Forza HTTPS

Proteggono da vulnerabilità comuni

🌐 CORS Headers

Access-Control-Allow-Origin: Domini permessi

Access-Control-Allow-Methods: Metodi consentiti

Access-Control-Allow-Headers: Headers accettati

Gestiscono accesso cross-origin

🚀 HTTP/2 vs HTTP/3: Le Rivoluzioni

🔄 HTTP/2 Innovations

Protocollo binario più efficiente del testo.

  • Multiplexing: Più richieste sulla stessa connessione
  • Server Push: Server invia risorse prima che siano richieste
  • Header Compression: HPACK riduce overhead
  • Stream Priority: Priorità delle richieste

⚡ HTTP/3 + QUIC

UDP-based per ridurre latenza drasticamente.

  • 0-RTT Handshake: Connessioni istantanee
  • Connection Migration: Sopravvive al cambio rete
  • Built-in Encryption: TLS integrato
  • Head-of-line Blocking: Risolto a livello transport

📈 Performance Impact

Miglioramenti misurabili in scenari reali.

  • HTTP/2: 10-30% più veloce su connessioni lente
  • HTTP/3: 20-50% riduzione latenza
  • Mobile Networks: Benefici maggiori
  • File Multipli: Parallelizzazione ottimale

⚙️ Considerazioni Implementation

Adozione graduale e compatibilità backward.

  • Server Support: Upgrade infrastructure richiesto
  • Proxy/CDN: Supporto ancora parziale
  • Debugging: Tool binari più complessi
  • Browser Support: HTTP/3 ancora in rollout

🔧 Best Practices HTTP

Performance: Usa HTTP/2+, abilita compressione, implementa caching intelligente.

Sicurezza: Sempre HTTPS in produzione, implementa security headers, valida input.

API Design: Rispetta semantica HTTP, usa status codes appropriati, implementa rate limiting.

Monitoring: Logga richieste, monitora latenza, traccia errori 4xx/5xx.

🏛️ Curiosità: HTTP/0.9

HTTP/0.9 aveva solo il metodo GET e supportava solo HTML. Nessun header, nessun status code - solo "dammi questa pagina"!

Una richiesta tipica: GET /index.html seguita dalla chiusura immediata della connessione. Primitivo ma funzionale!
